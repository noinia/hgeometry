smallestEnclosingDisk :: (Point_ point 2 r, Ord r, Fractional r, Foldable set) => set point -> DiskByPoints point Source #

Horrible \( O(n^4) \) implementation that simply tries all disks, checks if they enclose all points, and takes the largest one. Basically, this is only useful to check correctness of the other algorithm(s)

pre: the set contains at least 2 (distinct) points

smallestEnclosingDiskWith :: (Point_ point 2 r, Fractional r, Ord r, Foldable1 nonEmpty) => point -> nonEmpty point -> DiskByPoints point Source #

Given a point p and a non-empty set of n points, computes the smallest disk that has p on the boundary and encloses all points.

pre: such a disk exists

running time: \(O(n^2)\)
